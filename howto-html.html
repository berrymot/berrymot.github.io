document structure

here's the minimal requirement for a page:

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>very good page</title>
    </head>
    <body>
        <!-- whatever you want! -->
    </body>
</html>

you can save this to a file ending in .html if you want. if you then open it in a browser you'll see...

...oh that's not very much. here's what this does:

  • <!DOCTYPE html> is the doctype. it tells the browser this is html5, not some other version like 4.1, and magically prevents quirks in browsers.
  • <html> wraps the page. you might notice the </html> later: that's the end. the additional / to mark the end of something is the general pattern.
  • <head> contains information about the page:
    • <meta charset="utf-8" /> tells it to use unicode, because for some reason that still isn't the default. here the slash being at the end means it doesn't have any content or closing tag - there's no such thing as </meta>.
    • the "viewport" <meta /> makes it not super tiny on phones etc.
    • <title> gets put in the tab label - you'll notice not on the page itself.
  • <body>'s where you put the actual Stuff on the page!
  • <!-- ... --> wraps comments. these are just for you, they don't get put on the page.

you technically don't need to explicitly mark <html> or <head> or <body>, the browser will do it for you, but it's a good idea to anyway.
